# CRM in PC

Легка локальна CRM для персонального ПК. Перший реліз тепер орієнтований на **звичайний desktop-застосунок**: можна запускати через `.exe` на Windows або через Python-лаунчер під час розробки.

## Можливості першого релізу

- створення власних розділів CRM;
- реєстр записів відкривається одразу при вході в розділ;
- налаштування колонок реєстру: можна вибрати, які поля показувати в таблиці;
- контекстне меню запису в реєстрі: **відкрити**, **копіювати**, **видалити**;
- відкриття картки запису подвійним кліком по рядку;
- створення, редагування та видалення записів у картці;
- додавання полів прямо з картки або з вкладки “Поля розділу”;
- типи полів: текст, число, дата, багаторядковий текст, прапорець;
- сучасніші кольори, “glass”-панелі та легка прозорість вікна;
- локальне збереження даних у JSON-файлі на ПК: `~/.crm_in_pc/data.json`;
- імпорт та експорт бази у JSON;
- імпорт та експорт активного розділу в Excel `.xlsx`.

## Запуск як нормальний застосунок на Windows

### Варіант 1: готовий `.exe`

Після збірки файл буде тут:

```text
dist\CRM-in-PC.exe
```

Його можна запускати подвійним кліком як звичайну програму.

### Варіант 2: зібрати `.exe` самостійно

1. Встановіть Python 3 з [python.org](https://www.python.org/downloads/) і під час встановлення увімкніть **Add Python to PATH**.
2. Завантажте або розпакуйте цей репозиторій на компʼютер.
3. Запустіть подвійним кліком файл `build-windows-exe.bat`.
4. Після завершення відкрийте `dist\CRM-in-PC.exe`.

Скрипт використовує `CRM-in-PC.spec`, щоб зібрати віконний `.exe` без консолі.

### Варіант 3: запуск без збірки `.exe`

Якщо Python вже встановлено, можна запустити подвійним кліком:

```text
run-crm.bat
```

## macOS / Linux

```bash
./run-crm.sh
```

Або напряму:

```bash
python3 crm_desktop.py
```

## Excel

- **Експорт Excel** створює `.xlsx` для активного розділу: перший рядок — назви полів, нижче — записи.
- **Імпорт Excel** читає `.xlsx` і додає записи в активний розділ. Назви колонок мають збігатися з назвами полів у розділі.

## Вебверсія

Файли `index.html` та `src/app.js` залишені як легка вебверсія, але основний рекомендований сценарій для персонального ПК — `dist\CRM-in-PC.exe` або `run-crm.bat`.

## Тести

```bash
npm test
python3 -m unittest tests/test_crm_core.py
python3 -m py_compile crm_desktop.py src/crm_core.py src/excel_io.py
```

Тести перевіряють базові операції сховища, JSON-збереження, валідацію обовʼязкових полів та Excel `.xlsx` експорт/імпорт.
